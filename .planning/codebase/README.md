# Codebase Map

This folder contains a structured analysis of the Handy codebase, generated by
[Claude Code](https://claude.ai/code) using the
[get-shit-done](https://github.com/gsd-build/get-shit-done) skill pack.

Keeping this checked in helps AI coding assistants (Claude Code, Copilot, etc.)
onboard faster and produce better suggestions — no need to re-explore the whole
repo from scratch every session.

## Documents

| File | Contents |
|------|----------|
| STACK.md | Languages, frameworks, runtimes, build tools, package managers |
| INTEGRATIONS.md | External services, APIs, third-party libraries, protocols |
| ARCHITECTURE.md | System design, component relationships, data flow, patterns |
| STRUCTURE.md | Directory layout, file organization, module boundaries |
| CONVENTIONS.md | Code style, naming, i18n, error handling, TypeScript/Rust conventions |
| TESTING.md | Test setup, test types, coverage, how to run tests |
| CONCERNS.md | Tech debt, known issues, security considerations, platform quirks |

## Regenerating

Install [Claude Code](https://claude.ai/code) and the
[get-shit-done](https://github.com/gsd-build/get-shit-done) skill pack, then run:

```
/gsd:map-codebase
```

This spawns 4 parallel AI agents that explore the codebase and rewrite all 7
documents in under 5 minutes.
