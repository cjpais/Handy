# Architecture

**Analysis Date:** 2026-03-28

## Pattern Overview

**Overall:** Tauri 2.x desktop app — Rust backend + React/TypeScript frontend, communicating via Tauri's command/event IPC bridge.

**Key Characteristics:**
- Manager pattern: four long-lived Arc-wrapped managers owned by Tauri state
- Command-event architecture: frontend calls Rust commands; Rust emits events back
- Single-threaded coordinator: `TranscriptionCoordinator` serializes all pipeline lifecycle events to eliminate race conditions
- Action map: static `ACTION_MAP` dispatches shortcut binding IDs to `ShortcutAction` trait objects
- Two separate webview windows: `main` (settings UI) and `overlay` (recording status)

## Layers

**Frontend (React/TypeScript):**
- Purpose: Settings UI, onboarding, overlay window
- Location: `src/`
- Contains: React components, Zustand stores, hooks, i18n, bindings
- Depends on: Tauri JS APIs (`@tauri-apps/api`), auto-generated `src/bindings.ts`
- Used by: End user

**IPC Bridge:**
- Purpose: Type-safe frontend↔backend communication
- Location: `src/bindings.ts` (auto-generated by tauri-specta on debug builds)
- Contains: All command signatures and event types exported from Rust
- Pattern: `commands.someCommand(args)` on frontend; `#[tauri::command] #[specta::specta]` on backend

**Backend Commands (Rust):**
- Purpose: Thin handlers that delegate to managers or settings
- Location: `src-tauri/src/commands/`
- Submodules: `audio.rs`, `history.rs`, `models.rs`, `transcription.rs`, `mod.rs`
- Depends on: Managers, settings, portable
- Used by: Frontend via Tauri invoke

**Managers (Rust):**
- Purpose: Core stateful business logic, held as `Arc<T>` in Tauri state
- Location: `src-tauri/src/managers/`
- Contains:
  - `audio.rs` — microphone recording, device management, VAD integration, mute
  - `model.rs` — model download, deletion, load/unload lifecycle
  - `transcription.rs` — Whisper/Parakeet inference pipeline, accelerator config
  - `history.rs` — transcription history, WAV file retention, persistence
- Depends on: `audio_toolkit/`, settings, Tauri state

**TranscriptionCoordinator (Rust):**
- Purpose: Serializes all recording lifecycle events (keyboard, signals, CLI) through a single background thread via `mpsc::channel`
- Location: `src-tauri/src/transcription_coordinator.rs`
- States: `Idle → Recording(binding_id) → Processing → Idle`
- Prevents race conditions between concurrent shortcut presses, SIGUSR1/2, and `--toggle-transcription` CLI

**Actions (Rust):**
- Purpose: Implement the `ShortcutAction` trait; dispatched by `ACTION_MAP`
- Location: `src-tauri/src/actions.rs`
- Registered actions: `transcribe`, `transcribe_with_post_process`, `cancel`, `test`
- `TranscribeAction::stop` spawns the full async pipeline: stop recording → WAV save (concurrent) → transcribe → optional post-process → paste → history

**Audio Toolkit (Rust):**
- Purpose: Low-level audio: device enumeration, recording, resampling, VAD
- Location: `src-tauri/src/audio_toolkit/`
- Submodules: `audio/` (device, recorder, resampler, visualizer), `vad/` (Silero ONNX, smoothed)

**Settings (Rust):**
- Purpose: Persist/read `AppSettings` struct via `tauri-plugin-store`
- Location: `src-tauri/src/settings.rs`
- Pattern: `get_settings(app)` / `write_settings(app, settings)` — synchronous calls anywhere

## Data Flow

**Recording → Transcription → Paste Pipeline:**

```
User presses hotkey
        │
        ▼
shortcut/handler.rs → TranscriptionCoordinator.send_input()
        │
        ▼
TranscriptionCoordinator (background thread, mpsc)
        │  selects action from ACTION_MAP
        ▼
TranscribeAction::start()
  - AudioRecordingManager.try_start_recording()
  - play audio feedback (SoundType::Start)
  - show_recording_overlay() → emit "show-overlay" → RecordingOverlay.tsx
  - change_tray_icon(Recording)
        │
User releases hotkey / toggle
        │
        ▼
TranscribeAction::stop()
  - spawn async task
  - AudioRecordingManager.stop_recording() → Vec<f32> samples
  - [concurrent] save_wav_file()
  - TranscriptionManager.transcribe(samples) → String
  - [optional] post_process_transcription() via LLM API
  - HistoryManager.save_entry()
  - paste() on main thread (clipboard + keystroke)
  - hide_recording_overlay() → emit "hide-overlay"
  - change_tray_icon(Idle)
  - TranscriptionCoordinator.notify_processing_finished()
```

**Settings Update Flow:**

```
Settings component onChange
        │
        ▼
useSettings hook → settingsStore.updateSetting(key, value)
        │  optimistic update to Zustand store
        │
        ▼
settingUpdaters[key](value) → commands.someCommand(value) [bindings.ts]
        │
        ▼
Rust command handler → settings::write_settings()
        │  on error: Zustand rolls back to original value
```

**Backend → Frontend Events:**

| Event name            | Emitter                          | Consumer                        |
|-----------------------|----------------------------------|---------------------------------|
| `show-overlay`        | `utils::show_recording_overlay`  | `RecordingOverlay.tsx`          |
| `hide-overlay`        | `utils::hide_recording_overlay`  | `RecordingOverlay.tsx`          |
| `mic-level`           | `AudioRecordingManager`          | `RecordingOverlay.tsx`          |
| `recording-error`     | `actions.rs` (TranscribeAction)  | `App.tsx` (toast)               |
| `model-state-changed` | `managers/model.rs`              | `App.tsx` (toast), settingsStore|
| `history-update`      | `managers/history.rs`            | Frontend history components     |
| `check-for-updates`   | `lib.rs` tray / trigger command  | `UpdateChecker.tsx`             |

## Key Abstractions

**ShortcutAction trait:**
- Purpose: Decouples shortcut bindings from pipeline implementation
- Location: `src-tauri/src/actions.rs`
- Pattern: `ACTION_MAP: Lazy<HashMap<String, Arc<dyn ShortcutAction>>>` — static dispatch by binding ID string

**TranscriptionCoordinator:**
- Purpose: Single-owner pipeline state machine that prevents concurrent pipeline runs
- Location: `src-tauri/src/transcription_coordinator.rs`
- Pattern: Actor-style thread with `mpsc::Sender<Command>` — only one thread owns `Stage`

**Tauri Managed State:**
- All managers registered via `app_handle.manage(arc_manager)` in `initialize_core_logic()`
- Retrieved anywhere via `app.state::<Arc<ManagerType>>()`
- Managers: `Arc<AudioRecordingManager>`, `Arc<ModelManager>`, `Arc<TranscriptionManager>`, `Arc<HistoryManager>`, `TranscriptionCoordinator`, `CliArgs`, tray icon

**settingsStore (Zustand):**
- Location: `src/stores/settingsStore.ts`
- Pattern: `subscribeWithSelector` — optimistic updates with rollback on error
- `settingUpdaters` map: `keyof Settings → (value) => Promise<unknown>` — each setting key maps to its Rust command

**bindings.ts:**
- Location: `src/bindings.ts`
- Auto-generated by tauri-specta on debug builds from `#[specta::specta]` annotations
- Provides fully typed `commands.*` and event types to frontend

## Entry Points

**Backend startup:**
- Location: `src-tauri/src/main.rs` → `lib.rs::run(cli_args)`
- Sequence: `portable::init()` → plugin registration → `setup()` closure → `initialize_core_logic()` → tray init → show/hide main window

**Frontend main window:**
- Location: `src/main.tsx` → `src/App.tsx`
- Startup: checks onboarding status → accessibility/model steps → initializes Enigo + shortcuts after onboarding

**Frontend overlay window:**
- Location: `src/overlay/main.tsx` → `src/overlay/RecordingOverlay.tsx`
- Separate Tauri webview, listens for `show-overlay` / `hide-overlay` / `mic-level` events

**CLI remote control:**
- Location: `src-tauri/src/cli.rs`, `src-tauri/src/main.rs`
- Flags `--toggle-transcription`, `--toggle-post-process`, `--cancel` launch a second instance that routes through `tauri_plugin_single_instance` callback → `signal_handle::send_transcription_input()`

## Error Handling

**Strategy:** Explicit `Result<T, String>` on all Tauri commands; errors surfaced as toast notifications in the frontend

**Patterns:**
- Recording errors emitted as `recording-error` event with typed `error_type` field
- Model load failures emitted as `model-state-changed` with `event_type: "loading_failed"`
- Transcription failures save empty history entry (allowing retry) and reset UI state
- `FinishGuard` drop guard in `actions.rs` ensures `TranscriptionCoordinator` is always notified even on panic
- Settings updates: optimistic write + rollback pattern in `settingsStore.ts`

## Cross-Cutting Concerns

**Logging:** `tauri-plugin-log` with dual targets — stdout (RUST_LOG env filter) and rotating file (user-controlled level via `FILE_LOG_LEVEL` atomic); `--debug` CLI flag enables Trace

**Persistence:** `tauri-plugin-store` for settings; WAV files in `{app_data}/recordings/`; history metadata alongside WAV

**Portable mode:** `src-tauri/src/portable.rs` — if a portable marker file exists, redirects all data/log paths to a local `Data/` directory; detected at startup before anything else

**i18n:** `i18next` on frontend; `src/i18n/locales/{lang}/translation.json`; tray menu has its own i18n in `src-tauri/src/tray_i18n.rs`

**Post-processing:** Optional LLM step in `actions.rs::post_process_transcription()` — supports structured output (JSON schema) and legacy prompt replacement; `src-tauri/src/llm_client.rs` handles HTTP calls; Apple Intelligence on macOS ARM via `src-tauri/src/apple_intelligence.rs`
