# Dictate with Translation — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dedicated "Dictate with Translation" shortcut that transcribes speech and translates it into a user-chosen target language via the existing local-LLM post-processing pipeline, pasting only the translation.

**Architecture:** Introduce a `TranscribeMode` enum (`Plain`/`PostProcess`/`Translate`) on the existing `TranscribeAction`. The `Translate` mode reuses the post-processing LLM path (`llm_client`, provider/model/API-key settings) but drives it with a generated translation system prompt derived from a new `translation_target_language` setting. A new global shortcut binding maps to the `Translate` action. Frontend adds a target-language dropdown + shortcut input in the Post-Processing settings tab.

**Tech Stack:** Rust (Tauri 2, specta), TypeScript/React, Zustand, bun, i18next. LLM via existing `crate::llm_client` against an OpenAI-compatible local endpoint (Ollama/LM Studio, "custom" provider).

## Global Constraints

- Reuse the post-processing provider/model/API-key settings for translation — do NOT add a separate provider config.
- `translation_target_language` stores a language **code** (e.g. `"de"`), default `"en"`; the dropdown always holds a value (no "unset" state).
- New Rust setting field MUST use `#[serde(default = "...")]` for backward-compatible load of existing `settings_store.json`.
- All new user-facing frontend strings go through i18next (`src/i18n/locales/en/translation.json`); no hardcoded JSX strings (ESLint `eslint-plugin-i18next` will fail the build otherwise).
- Build with `export CARGO_TARGET_DIR="C:/h"` on Windows (260-char path limit); the app is a Tauri app built via `bun tauri dev` / `bun tauri build`.
- Rust settings-change commands live in `src-tauri/src/shortcut/mod.rs` and MUST be registered in the `collect_commands!`/`invoke_handler` list in `src-tauri/src/lib.rs`.
- `src/bindings.ts` is auto-generated by tauri-specta at build time — never hand-edit it; rebuild to regenerate after changing Rust command/type signatures.

```rust
/// English display name for a language code, mirroring the common entries of the
/// frontend `LANGUAGES` list (src/lib/constants/languages.ts). Only the widely
/// used targets are mapped; an unknown code falls back to the code itself, which
/// LLMs still interpret acceptably.
fn language_english_name(code: &str) -> String {
    let name = match code {
        "en" => "English",
        "zh" | "zh-Hans" => "Simplified Chinese",
        "zh-Hant" => "Traditional Chinese",
        "es" => "Spanish",
        "fr" => "French",
        "de" => "German",
        "ja" => "Japanese",
        "ko" => "Korean",
        "ru" => "Russian",
        "pt" => "Portuguese",
        "it" => "Italian",
        "nl" => "Dutch",
        "pl" => "Polish",
        "tr" => "Turkish",
        "uk" => "Ukrainian",
        "ar" => "Arabic",
        "he" => "Hebrew",
        "hi" => "Hindi",
        "sv" => "Swedish",
        "cs" => "Czech",
        "el" => "Greek",
        "ro" => "Romanian",
        "hu" => "Hungarian",
        "fi" => "Finnish",
        "da" => "Danish",
        "no" => "Norwegian",
        "vi" => "Vietnamese",
        "th" => "Thai",
        "id" => "Indonesian",
        _ => return code.to_string(),
    };
    name.to_string()
}

/// Build the translation system prompt for the given target language code.
fn build_translation_prompt(target_language_code: &str) -> String {
    let language = language_english_name(target_language_code);
    format!(
        "Translate the following text into {language}. Output only the translation, \
         preserving the original meaning and tone. Do not add explanations, notes, \
         or quotation marks."
    )
}
```

Replace the `if post_process { ... } else if final_text != transcription { ... }` block with:

```rust
    match mode {
        TranscribeMode::PostProcess => {
            let system_prompt = settings
                .post_process_selected_prompt_id
                .as_ref()
                .and_then(|id| {
                    settings
                        .post_process_prompts
                        .iter()
                        .find(|p| &p.id == id)
                })
                .map(|p| p.prompt.clone());

            if let Some(system_prompt) = system_prompt {
                if let Some(processed) = run_llm(&settings, &final_text, system_prompt.clone()).await
                {
                    post_processed_text = Some(processed.clone());
                    final_text = processed;
                    post_process_prompt = Some(system_prompt);
                }
            }
        }
        TranscribeMode::Translate => {
            let system_prompt = build_translation_prompt(&settings.translation_target_language);
            if let Some(processed) = run_llm(&settings, &final_text, system_prompt.clone()).await {
                post_processed_text = Some(processed.clone());
                final_text = processed;
                post_process_prompt = Some(system_prompt);
            }
        }
        TranscribeMode::Plain => {
            if final_text != transcription {
                post_processed_text = Some(final_text.clone());
            }
        }
    }
```
